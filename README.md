## Callie: Chosen Scenario and Four Additional Elements

Our team has chosen the third-person game scenario. We are planning to make a single-level horror hotel game with a clear starting area, a final goal area, and at least five rooms of different sizes.

The four additional elements we chose are:

- Player Power-Up Pickups: Health kits, flashlight batteries, and other helpful items that can be used for survival or puzzles.
- Enemies (Stationary): Haunted hotel objects like suitcases, portraits, lamps, and the front desk bell that can damage or affect the player.
- Enemies (Moving): Haunted bellhops, hotel staff, or guests that move around the hotel and can chase or attack the player.
- Enemies (Bosses): A hotel manager boss who acts as the final major enemy encounter.

## Kay: Developmental Goals
- For our Alpha build, we will focus on making sure the third-person player works, the hotel level is blocked out, the starting area and goal area are in, one pickup works, one enemy or haunted object works, and having at least a basic HUD implemented.
- For our Beta build, our goal is to have the level playable from start to finish. This means having most of the pickups, enemies, boss/progression mechanics, UI, and hotel visuals working. This will be the version where we test everything, fix major issues, and figure out what still needs polish before the final release.

## August: Development Schedule and Timeline

Our team will follow the course due dates to keep the project on track. We will focus on building a small playable version first, then adding more features and polish as we move through the Alpha, Beta, and Final releases

### Module Two: Project Plan
- Finalize the game idea, scenario, roles, communication plan, and four additional elements
- Set up the GitHub repository and README
- Keep the scope focused on the smallest playable version first

### Module Three: QA and Testing Plan
- Create the testing plan
- Decide how bugs and issues will be tracked
- Start planning the hotel layout and core gameplay loop
- Define the minimum playable version of the game

### Module Four: Alpha Release
- Build the basic playable version of the game
- Implement basic third-person player movement
- Block out the hotel level and main rooms
- Add the starting area and goal area
- Add at least one working pickup
- Add at least one working enemy or hazard
- Start basic UI/HUD elements

### Module Five: Beta Release
- Make the level playable from start to finish
- Add more of the required gameplay elements
- Add progression mechanics, such as keycards or locked areas, if used
- Make the UI mostly functional
- Begin testing and fixing major issues
- Add more hotel/horror visuals, textures, and assets

### Module Six: Final Level Release
- Finish all required features
- Polish the level, UI, textures, and gameplay flow
- Fix bugs found during testing
- Make sure the level is fully playable
- Make sure the level meets the 5–10 minute playtime goal
